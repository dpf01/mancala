* CONTEXT.md provides developer context for the project in this directory. Make sure it is kept up to date as code changes are made.
* README.md provides information to a user of the code: what it does (briefly), how to compile and run it, what flags are available.
* todo.md provides instructions on the current task. It is cleared out for each task, so any information worth persisting should be added to one of the other markdown files. This file should not be modified by the agent and is never committed to source control.
* Code comments and markdown text should always reflect the current state of the program, written as if the program has always worked that way. They should not say things like "(new)" or "Update #2: make the foo green instead of blue".
